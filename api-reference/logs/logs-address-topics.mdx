title: "Get Event Logs by Address and Topics Combined"
sidebarTitle: "Logs by Address + Topics"

---

Retrieve event logs emitted by a specific contract address, with additional filtering by one or more topic values and a block range. This is the most precise form of log query — combining address restriction with topic filtering lets you target exactly the event type you're interested in from a particular contract. Use this endpoint when you need to track specific events like sales, transfers, or swaps from a known contract.

## Query Parameters

<ParamField query="apikey" type="string" required>
  Your Etherscan API key.
</ParamField>

<ParamField query="chainid" type="string" default="1">
  Chain ID to query, e.g. `1` for Ethereum, `8453` for Base. See the full list of [supported chains](/supported-chains).
</ParamField>

<ParamField query="module" type="string" required>
  Set to `logs` for this endpoint.
</ParamField>

<ParamField query="action" type="string" required>
  Set to `getLogs` for this endpoint.
</ParamField>

<ParamField query="address" type="string" required>
  The contract address to retrieve logs from. Example: `0x59728544b08ab483533076417fbbb2fd0b17ce3a`.
</ParamField>

<ParamField query="fromBlock" type="integer" required>
  The starting block number (inclusive) to search from. Example: `15073139`.
</ParamField>

<ParamField query="toBlock" type="integer" required>
  The ending block number (inclusive) to search to. Example: `15074139`.
</ParamField>

<ParamField query="topic0" type="string">
  The first topic to filter by, typically the keccak-256 hash of an event signature. Example: `0x27c4f0403323142b599832f26acd21c74a9e5b809f2215726e244a4ac588cd7d`.
</ParamField>

<ParamField query="topic0_1_opr" type="string" default="and">
  The logical operator applied between `topic0` and `topic1`. Use `and` to require both, or `or` to match either.
</ParamField>

<ParamField query="topic1" type="string">
  The second topic to filter by. Example: `0x00000000000000000000000023581767a106ae21c074b2276d25e5c3e136a68b`.
</ParamField>

<ParamField query="page" type="integer" default="1">
  Page number for pagination.
</ParamField>

<ParamField query="offset" type="integer" default="100">
  Number of records per page.
</ParamField>

## Response Fields

<ResponseField name="status" type="string">
  `"1"` on success, `"0"` on failure.
</ResponseField>

<ResponseField name="message" type="string">
  `"OK"` on success.
</ResponseField>

<ResponseField name="result" type="array">
  An array of log entry objects matching both the address and topic filters.

  <Expandable title="log entry fields">
    <ResponseField name="address" type="string">
      The contract address that emitted the log.
    </ResponseField>

    <ResponseField name="topics" type="array">
      An array of up to four indexed topic values for this log entry.
    </ResponseField>

    <ResponseField name="data" type="string">
      The ABI-encoded non-indexed event parameters as a hex string.
    </ResponseField>

    <ResponseField name="blockNumber" type="string">
      The block number in which the log was emitted, as a hex string.
    </ResponseField>

    <ResponseField name="blockHash" type="string">
      The hash of the block containing this log.
    </ResponseField>

    <ResponseField name="timeStamp" type="string">
      The Unix timestamp of the block, as a hex string.
    </ResponseField>

    <ResponseField name="gasPrice" type="string">
      The gas price of the transaction, as a hex string in wei.
    </ResponseField>

    <ResponseField name="gasUsed" type="string">
      Total gas used by the transaction, as a hex string.
    </ResponseField>

    <ResponseField name="logIndex" type="string">
      The position of this log within the block, as a hex string.
    </ResponseField>

    <ResponseField name="transactionHash" type="string">
      The hash of the transaction that emitted this log.
    </ResponseField>

    <ResponseField name="transactionIndex" type="string">
      The position of the transaction within the block, as a hex string.
    </ResponseField>
  </Expandable>
</ResponseField>

## Example Request

This example retrieves logs from an NFT marketplace contract filtered by a specific event signature and a buyer address over a short block window.

```bash
curl "https://api.etherscan.io/v2/api\
?chainid=1\
&module=logs\
&action=getLogs\
&fromBlock=15073139\
&toBlock=15074139\
&address=0x59728544b08ab483533076417fbbb2fd0b17ce3a\
&topic0=0x27c4f0403323142b599832f26acd21c74a9e5b809f2215726e244a4ac588cd7d\
&topic0_1_opr=and\
&topic1=0x00000000000000000000000023581767a106ae21c074b2276d25e5c3e136a68b\
&page=1\
&offset=100\
&apikey=YourApiKeyToken"
```
