These endpoints let you build a complete token portfolio view for any address. You can fetch all ERC-20 tokens held with their current USD values, all ERC-721 NFT collections with holding counts, or a detailed inventory of specific token IDs owned within a given NFT contract. All three endpoints require a Standard plan or above.

---

## ERC-20 Token Holdings

<Warning>
  This is a PRO endpoint available to the Standard plan and above. This endpoint is also throttled to **2 calls/second** regardless of API Pro tier.
</Warning>

Retrieves the current ERC-20 token holdings for a specified address, including token balances and their current USD values. Results are paginated — use `page` and `offset` to iterate through all holdings.

### Query Parameters

<ParamField query="apikey" type="string" default="YourApiKeyToken">
  Your Etherscan API key.
</ParamField>

<ParamField query="chainid" type="string" default="1">
  Chain ID to query, e.g. `1` for Ethereum, `8453` for Base from our [supported chains](/supported-chains).
</ParamField>

<ParamField query="module" type="string" default="account">
  Set to `account` for this endpoint.
</ParamField>

<ParamField query="action" type="string" default="addresstokenbalance">
  Set to `addresstokenbalance` for this endpoint.
</ParamField>

<ParamField query="address" type="string" default="0x983e3660c0bE01991785F80f266A84B911ab59b0">
  Address to check for token holdings.
</ParamField>

<ParamField query="page" type="integer" default="1">
  Page number for pagination.
</ParamField>

<ParamField query="offset" type="integer" default="100">
  Number of records per page. Use the `page` parameter for subsequent records.
</ParamField>

### Example Request

```bash
curl "https://api.etherscan.io/v2/api\
?chainid=1\
&module=account\
&action=addresstokenbalance\
&address=0x983e3660c0bE01991785F80f266A84B911ab59b0\
&page=1\
&offset=100\
&apikey=YourApiKeyToken"
```

### Response

<ResponseField name="status" type="string">
  `"1"` on success, `"0"` on failure.
</ResponseField>

<ResponseField name="message" type="string">
  `"OK"` on success, or an error description on failure.
</ResponseField>

<ResponseField name="result" type="array">
  Array of ERC-20 token holding objects.

  <ResponseField name="TokenAddress" type="string">
    Contract address of the ERC-20 token.
  </ResponseField>

  <ResponseField name="TokenName" type="string">
    Full name of the token, e.g. `"Furucombo"`.
  </ResponseField>

  <ResponseField name="TokenSymbol" type="string">
    Ticker symbol of the token, e.g. `"COMBO"`.
  </ResponseField>

  <ResponseField name="TokenQuantity" type="string">
    Token balance held by the address, in the token's smallest unit.
  </ResponseField>

  <ResponseField name="TokenDivisor" type="string">
    Number of decimal places for the token. Divide `TokenQuantity` by `10^TokenDivisor` for a human-readable value.
  </ResponseField>

  <ResponseField name="TokenPriceUSD" type="string">
    Current USD price per token. May be `"0.000000000000000000"` if not tracked.
  </ResponseField>
</ResponseField>

## ERC-721 NFT Holdings

<Warning>
  This is a PRO endpoint available to the Standard plan and above. This endpoint is also throttled to **2 calls/second** regardless of API Pro tier.
</Warning>

Retrieves the ERC-721 NFT collections and their quantities owned by a specific address. Each result represents one NFT collection (contract), along with the number of tokens the address holds from that collection. Results are paginated — use `page` and `offset` to iterate.

### Query Parameters

<ParamField query="apikey" type="string" default="YourApiKeyToken">
  Your Etherscan API key.
</ParamField>

<ParamField query="chainid" type="string" default="1">
  Chain ID to query, e.g. `1` for Ethereum, `8453` for Base from our [supported chains](/supported-chains).
</ParamField>

<ParamField query="module" type="string" default="account">
  Set to `account` for this endpoint.
</ParamField>

<ParamField query="action" type="string" default="addresstokennftbalance">
  Set to `addresstokennftbalance` for this endpoint.
</ParamField>

<ParamField query="address" type="string" default="0x6b52e83941eb10f9c613c395a834457559a80114">
  Address to check for NFT holdings.
</ParamField>

<ParamField query="page" type="integer" default="1">
  Page number for pagination.
</ParamField>

<ParamField query="offset" type="integer" default="100">
  Number of records per page. Use the `page` parameter for subsequent records.
</ParamField>

### Example Request

```bash
curl "https://api.etherscan.io/v2/api\
?chainid=1\
&module=account\
&action=addresstokennftbalance\
&address=0x6b52e83941eb10f9c613c395a834457559a80114\
&page=1\
&offset=100\
&apikey=YourApiKeyToken"
```

### Response

<ResponseField name="status" type="string">
  `"1"` on success, `"0"` on failure.
</ResponseField>

<ResponseField name="message" type="string">
  `"OK"` on success, or an error description on failure.
</ResponseField>

<ResponseField name="result" type="array">
  Array of ERC-721 NFT collection holding objects.

  <ResponseField name="TokenAddress" type="string">
    Contract address of the NFT collection.
  </ResponseField>

  <ResponseField name="TokenName" type="string">
    Name of the NFT collection.
  </ResponseField>

  <ResponseField name="TokenSymbol" type="string">
    Ticker symbol of the NFT collection.
  </ResponseField>

  <ResponseField name="TokenQuantity" type="string">
    Number of NFTs from this collection held by the address.
  </ResponseField>
</ResponseField>

<ResponseExample>
```json Response

  "status": "0",
  "message": "NOTOK",
  "result": "Maximum rate limit reached"

```
</ResponseExample>

## ERC-721 NFT Inventory by Contract

<Warning>
  This is a PRO endpoint available to the Standard plan and above. This endpoint is also throttled to **2 calls/second** regardless of API Pro tier.
</Warning>

Retrieves the specific ERC-721 token IDs owned by a given address within a particular NFT contract. Use this endpoint to enumerate the exact NFTs an address holds in a known collection. Results are paginated — use `page` and `offset` to iterate through the full inventory.

### Query Parameters

<ParamField query="apikey" type="string" default="YourApiKeyToken">
  Your Etherscan API key.
</ParamField>

<ParamField query="chainid" type="string" default="1">
  Chain ID to query, e.g. `1` for Ethereum, `8453` for Base from our [supported chains](/supported-chains).
</ParamField>

<ParamField query="module" type="string" default="account">
  Set to `account` for this endpoint.
</ParamField>

<ParamField query="action" type="string" default="addresstokennftinventory">
  Set to `addresstokennftinventory` for this endpoint.
</ParamField>

<ParamField query="address" type="string" default="0x123432244443b54409430979df8333f9308a6040">
  Address to check for inventory.
</ParamField>

<ParamField query="contractaddress" type="string" default="0xed5af388653567af2f388e6224dc7c4b3241c544">
  ERC-721 token contract address to filter by.
</ParamField>

<ParamField query="page" type="integer" default="1">
  Page number for pagination.
</ParamField>

<ParamField query="offset" type="integer" default="100">
  Number of records per page. Use the `page` parameter for subsequent records.
</ParamField>

### Example Request

```bash
curl "https://api.etherscan.io/v2/api\
?chainid=1\
&module=account\
&action=addresstokennftinventory\
&address=0x123432244443b54409430979df8333f9308a6040\
&contractaddress=0xed5af388653567af2f388e6224dc7c4b3241c544\
&page=1\
&offset=100\
&apikey=YourApiKeyToken"
```

### Response

<ResponseField name="status" type="string">
  `"1"` on success, `"0"` on failure.
</ResponseField>

<ResponseField name="message" type="string">
  `"OK"` on success, or an error description on failure.
</ResponseField>

<ResponseField name="result" type="array">
  Array of NFT inventory objects, one per token ID owned.

  <ResponseField name="TokenAddress" type="string">
    Contract address of the NFT collection.
  </ResponseField>

  <ResponseField name="TokenId" type="string">
    The specific ERC-721 token ID owned by the address within this contract.
  </ResponseField>
</ResponseField>

<ResponseExample>
```json Response
{
  "status": "1",
  "message": "OK",
  "result": [
    {
      "TokenAddress": "0xed5af388653567af2f388e6224dc7c4b3241c544",
      "TokenId": "453"
    },
    {
      "TokenAddress": "0xed5af388653567af2f388e6224dc7c4b3241c544",
      "TokenId": "8160"
    }
  ]
}
```
</ResponseExample>
