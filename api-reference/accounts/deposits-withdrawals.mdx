Etherscan provides several account endpoints for querying cross-chain and protocol-level fund movements. This page covers four related endpoints: L1-to-L2 deposit transactions, L2-to-L1 withdrawal transactions, Ethereum beacon chain withdrawal transactions, and Plasma bridge deposits. Each section below documents the relevant endpoint, its parameters, and an example response.

---

## Deposit Transactions

The `getdeposittxs` action retrieves all cross-chain deposit transactions sent from Ethereum (L1) to an L2 network for a specified address. This endpoint is available on Arbitrum, Optimism, and their testnets.

Available on: **Arbitrum One Mainnet**, **Arbitrum Sepolia Testnet**, **OP Mainnet**, **OP Sepolia Testnet**

### Request

```bash
curl "https://api.etherscan.io/v2/api\
?chainid=10\
&module=account\
&action=getdeposittxs\
&address=0x80f3950a4d371c43360f292a4170624abd9eed03\
&page=1\
&offset=100\
&sort=asc\
&apikey=YourApiKeyToken"
```

### Query Parameters

<ParamField query="apikey" type="string" default="YourApiKeyToken" required>
  Your Etherscan API key.
</ParamField>

<ParamField query="chainid" type="string" default="10" required>
  Chain ID to query, e.g. `10` for Optimism or `42161` for Arbitrum. See the full list at [supported chains](/api-reference/chainlist).
</ParamField>

<ParamField query="module" type="string" default="account" required>
  Set to `account` for this endpoint.
</ParamField>

<ParamField query="action" type="string" default="getdeposittxs" required>
  Set to `getdeposittxs` for this endpoint.
</ParamField>

<ParamField query="address" type="string" default="0x80f3950a4d371c43360f292a4170624abd9eed03" required>
  Address to check for cross-chain deposits from Ethereum to L2.
</ParamField>

<ParamField query="page" type="integer" default="1">
  Page number for pagination.
</ParamField>

<ParamField query="offset" type="integer" default="100">
  Number of records per page. Use the `page` parameter for subsequent records.
</ParamField>

<ParamField query="sort" type="string" default="asc">
  Sort order: use `desc` to return the latest transactions first, or `asc` to return the oldest transactions first.
</ParamField>

## Withdrawal Transactions

The `getwithdrawaltxs` action retrieves all cross-chain withdrawal transactions sent from an L2 network back to Ethereum (L1) for a specified address. This endpoint is available on Arbitrum, Optimism, and their testnets.

Available on: **Arbitrum One Mainnet**, **Arbitrum Sepolia Testnet**, **OP Mainnet**, **OP Sepolia Testnet**

### Request

```bash
curl "https://api.etherscan.io/v2/api\
?chainid=10\
&module=account\
&action=getwithdrawaltxs\
&address=0x80f3950a4d371c43360f292a4170624abd9eed03\
&page=1\
&offset=100\
&sort=asc\
&apikey=YourApiKeyToken"
```

### Query Parameters

<ParamField query="apikey" type="string" default="YourApiKeyToken" required>
  Your Etherscan API key.
</ParamField>

<ParamField query="chainid" type="string" default="10" required>
  Chain ID to query, e.g. `10` for Optimism or `42161` for Arbitrum. See the full list at [supported chains](/api-reference/chainlist).
</ParamField>

<ParamField query="module" type="string" default="account" required>
  Set to `account` for this endpoint.
</ParamField>

<ParamField query="action" type="string" default="getwithdrawaltxs" required>
  Set to `getwithdrawaltxs` for this endpoint.
</ParamField>

<ParamField query="address" type="string" default="0x80f3950a4d371c43360f292a4170624abd9eed03" required>
  Address to check for cross-chain withdrawals from L2 to Ethereum.
</ParamField>

<ParamField query="page" type="integer" default="1">
  Page number for pagination.
</ParamField>

<ParamField query="offset" type="integer" default="100">
  Number of records per page. Use the `page` parameter for subsequent records.
</ParamField>

<ParamField query="sort" type="string" default="asc">
  Sort order: use `desc` to return the latest transactions first, or `asc` to return the oldest transactions first.
</ParamField>

### Response Example

```json
{
  "status": "0",
  "message": "No transactions found",
  "result": []
}
```

<Note>
  A `status` of `"0"` with `"No transactions found"` is a normal response for addresses that have not initiated any L2-to-L1 withdrawals. This is not an error.
</Note>

## Beacon Chain Withdrawals

The `txsBeaconWithdrawal` action retrieves Ethereum beacon chain withdrawal transactions made to a specified address. These are the automatic ETH withdrawals from Ethereum validators that occur via the consensus layer, as introduced in the Shapella upgrade (EIP-4895).

<Warning>
  **Effective July 1, 2026**, the maximum records returned per request will be reduced from 10,000 to **1,000** for Free tier API users. See the full list of [affected endpoints](https://docs.etherscan.io/changelog#upcoming-change-reduced-maximum-records-per-request-on-the-free-api-tier).
</Warning>

### Request

```bash
curl "https://api.etherscan.io/v2/api\
?chainid=1\
&module=account\
&action=txsBeaconWithdrawal\
&address=0xB9D7934878B5FB9610B3fE8A5e441e8fad7E293f\
&startblock=0\
&endblock=999999999\
&page=1\
&offset=100\
&sort=asc\
&apikey=YourApiKeyToken"
```

### Query Parameters

<ParamField query="apikey" type="string" default="YourApiKeyToken" required>
  Your Etherscan API key.
</ParamField>

<ParamField query="chainid" type="string" default="1" required>
  Chain ID to query, e.g. `1` for Ethereum. See the full list at [supported chains](/api-reference/chainlist).
</ParamField>

<ParamField query="module" type="string" default="account" required>
  Set to `account` for this endpoint.
</ParamField>

<ParamField query="action" type="string" default="txsBeaconWithdrawal" required>
  Set to `txsBeaconWithdrawal` for this endpoint.
</ParamField>

<ParamField query="address" type="string" default="0xB9D7934878B5FB9610B3fE8A5e441e8fad7E293f" required>
  The withdrawal credentials address to check for beacon chain withdrawals.
</ParamField>

<ParamField query="startblock" type="integer" default="0">
  Starting block number to search from.
</ParamField>

<ParamField query="endblock" type="integer" default="999999999">
  Ending block number to search to.
</ParamField>

<ParamField query="page" type="integer" default="1">
  Page number for pagination.
</ParamField>

<ParamField query="offset" type="integer" default="100">
  Number of records per page. Use the `page` parameter for subsequent records.
</ParamField>

<ParamField query="sort" type="string" default="asc">
  Sort order: use `desc` to return the latest withdrawals first, or `asc` to return the oldest first.
</ParamField>

### Response Fields

<ResponseField name="result[].withdrawalIndex" type="string">
  The global sequential index of this withdrawal across all validators.
</ResponseField>

<ResponseField name="result[].validatorIndex" type="string">
  The index of the validator that generated this withdrawal.
</ResponseField>

<ResponseField name="result[].address" type="string">
  The execution layer address that received the withdrawal.
</ResponseField>

<ResponseField name="result[].amount" type="string">
  The withdrawal amount in Gwei (1 ETH = 10^9 Gwei).
</ResponseField>

<ResponseField name="result[].blockNumber" type="string">
  The block number in which the withdrawal was processed.
</ResponseField>

<ResponseField name="result[].timestamp" type="string">
  Unix timestamp of the block.
</ResponseField>

### Response Example

```json
{
  "status": "1",
  "message": "OK",
  "result": [
    {
      "withdrawalIndex": "13",
      "validatorIndex": "117823",
      "address": "0xb9d7934878b5fb9610b3fe8a5e441e8fad7e293f",
      "amount": "3402931175",
      "blockNumber": "17034877",
      "timestamp": "1681338599"
    }

}
```

## Plasma Deposits

The `txnbridge` action retrieves Plasma bridge deposit transactions received by a specified address. Plasma deposits represent assets bridged from Ethereum to a Plasma-compatible chain such as Polygon (PoS), Gnosis, or BitTorrent Chain.

Available on: **Polygon Mainnet**, **Gnosis**, **BitTorrent Chain Mainnet**

<Warning>
  **Effective July 1, 2026**, the maximum records returned per request will be reduced from 10,000 to **1,000** for Free tier API users. See the full list of [affected endpoints](https://docs.etherscan.io/changelog#upcoming-change-reduced-maximum-records-per-request-on-the-free-api-tier).
</Warning>

### Request

```bash
curl "https://api.etherscan.io/v2/api\
?chainid=137\
&module=account\
&action=txnbridge\
&address=0x4880bd4695a8e59dc527d124085749744b6c988e\
&page=1\
&offset=100\
&apikey=YourApiKeyToken"
```

### Query Parameters

<ParamField query="apikey" type="string" default="YourApiKeyToken" required>
  Your Etherscan API key.
</ParamField>

<ParamField query="chainid" type="string" default="137" required>
  Chain ID to query, e.g. `137` for Polygon Mainnet. See the full list at [supported chains](/api-reference/chainlist).
</ParamField>

<ParamField query="module" type="string" default="account" required>
  Set to `account` for this endpoint.
</ParamField>

<ParamField query="action" type="string" default="txnbridge" required>
  Set to `txnbridge` for this endpoint.
</ParamField>

<ParamField query="address" type="string" default="0x4880bd4695a8e59dc527d124085749744b6c988e" required>
  The address to check for incoming Plasma bridge deposits.
</ParamField>

<ParamField query="page" type="integer" default="1">
  Page number for pagination.
</ParamField>

<ParamField query="offset" type="integer" default="100">
  Number of records per page. Use the `page` parameter for subsequent records.
</ParamField>

### Response Fields

<ResponseField name="result[].hash" type="string">
  The transaction hash of the Plasma deposit on the L2 chain.
</ResponseField>

<ResponseField name="result[].blockNumber" type="string">
  The block number in which the deposit was included.
</ResponseField>

<ResponseField name="result[].timeStamp" type="string">
  Unix timestamp of the block.
</ResponseField>

<ResponseField name="result[].from" type="string">
  The sender address (typically the zero address `0x000...000` for system-generated deposits).
</ResponseField>

<ResponseField name="result[].address" type="string">
  The recipient address that received the bridged tokens.
</ResponseField>

<ResponseField name="result[].amount" type="string">
  The amount of tokens deposited in the token's smallest unit.
</ResponseField>

<ResponseField name="result[].tokenName" type="string">
  The full name of the bridged token (e.g. `"Polygon Token"`).
</ResponseField>

<ResponseField name="result[].symbol" type="string">
  The ticker symbol of the bridged token (e.g. `"POL"`).
</ResponseField>

<ResponseField name="result[].contractAddress" type="string">
  The contract address of the token on the L2 chain.
</ResponseField>

<ResponseField name="result[].divisor" type="string">
  The number of decimal places for the token (e.g. `"18"` for 18-decimal tokens).
</ResponseField>
