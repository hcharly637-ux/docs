[Foundry](https://book.getfoundry.sh/) is a tool that helps take the heat off smart contract development steps, including compiling, deploying, and finally submitting your contract for verification. This guide assumes you already have a [Foundry project](https://getfoundry.sh/guides/project-setup/creating-a-new-project) set up and are ready to deploy and verify next.

<Info>
  Ensure you are using the latest version of Foundry by running `foundryup` before following this guide.
</Info>

## Deploy and Verify

Deploy your contract and verify it on Etherscan in a single `forge create` command by passing the `--verify` flag along with your Etherscan API key.

```bash
forge create --broadcast --rpc-url https://rpc.sepolia.ethpandaops.io --private-key YourPrivateKey src/ContractFile.sol:ContractName --verify --verifier etherscan --etherscan-api-key YourApiKeyToken
```

## Verify an Existing Contract

If you have already deployed a contract and need to verify it separately, use `forge verify-contract` with the deployed address, source path, and your API key.

```bash
forge verify-contract --watch --chain sepolia 0x324eca20b358b18e48f2611f7452560ce3b3c1bb src/ContractFile.sol:ContractName --verifier etherscan --etherscan-api-key YourApiKeyToken
```

## Custom Chains

Most chains with an Etherscan explorer are supported using the `--chain` flag. For very new chains that are not yet natively supported by Foundry, you can provide a custom `--verifier-url` instead.

Append the Etherscan V2 base path and the new `chainId` as a query parameter. The example below shows verification on Stable Testnet (chain ID `2201`).

```bash
forge verify-contract 0x324eca20b358b18e48f2611f7452560ce3b3c1bb src/ContractFile.sol:ContractName --verifier etherscan --verifier-url "https://api.etherscan.io/v2/api?chainid=2201" --etherscan-api-key YourEtherscanApiKey --watch
```
